# Code of Conduct

## Our pledge

We are committed to a friendly, respectful, and practical open-source community.

## Expected behavior

- Be respectful and constructive.
- Prefer minimal, reproducible examples.
- Do not post real secrets, private repository contents, customer data, or credentials.
- Assume good intent while keeping security boundaries clear.

## Unacceptable behavior

- Harassment, threats, or personal attacks.
- Publishing another person's private information.
- Posting live credentials or private data in issues, discussions, pull requests, or examples.
- Using the project to intentionally exfiltrate data.

## Reporting

For community issues, open a GitHub issue with a clear description.

For security issues, use GitHub Security Advisories once the repository is published. Do not open public issues for vulnerabilities that include real sensitive data.
